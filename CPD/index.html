    Designing for different size screens

Screen size and screen resolution -Desktop 1024 pixels width and above - 1024+, Tablet 1023 - 768, Mobile 767 – 320. Page sizes -Width 1000px - length 570px.
    Above the fold - the part of the page that will appear before the user can scroll. The first thing the user/reader will see. Inline styles is where you write the CSS directly into a style attribute on the HTML. Anything that appears above the fold you should use in line styles for. Anything else that does not appear above the fold you should not use in line styles for. Everything else should be inside of an external style sheet - from a technical point of view everything should be inside of an external style sheet.     CSS flexbox
    The flexbox layout was created to provide a more efficient way to layout, align and distribute spacing among items in a container, even when their sizing is unknown and/ or dynamic (thus the word ‘flex’).
    There is CSS specific to flexbox. Once a parent element has been told to display flex, you have a container, inside of the container you have items, you can then tell the items if they need to be specific widths etc. You have cross axis and a main axis, a main size and a cross size. CSS before flexbox meant everything just stacked on top of itself in html, CSS with flexbox - containers can be displayed next to one another.

Intro. to JavaScript

What have I learnt?

I was provided with a more in-depth insight into what JavaScript does, why and how. Also touched on the importance of pseudocode. Again, I have added the notes I made during the session to show what I understood/ took from the session.

    How do computers fit in with the world around us?
Objects contains data, object could be a person and the variables could be colour-brown, jumper-blue etc. Events are things that can happen- often referred to as triggers. Event triggers functions. Events could be when the mouse double clicks something on the page and an event occurs, hovers over something and something happens. JavaScript language has native types of objects. In JavaScript – an Array can only have numbers as keys.
    The DOM – Document Object Model
How a browser sees a webpage: - Receives a page as HTML code first – Each page on a website can be seen as a separate document. - Create a model of the page and store it in memory – this is called DOM - Use a rendering engine to show the page on the screen – there is no CSS at this stage
    What is JavaScript? What it can do, and using it with HTML
JavaScript is NOT JAVA. It is a Client Side scripting language. It works in your browser and most recently works on servers. HTML is the first layer of a web page – describes and provides structure. CSS is the second layer of a web page which is for presentation and layout. JavaScript is the third layer of a web page which is for functionality. What can JS do? Anything active, anything that changes, if you do something and something else happens, that’s JS. Dropdown boxes/ carousels. Lightboxes/modals. Drawing/ animation. Track users and hold data with cookies. Cookies are little bits of data that are stored on your device so that it makes it easier to log in next time basically. Cookies have a shelf life. Cookies are useful. Not tracking not kept on cookies.
    Pseudocode
Pseudocode – helps with algorithms. Set of instructions and guidance as to what the programme needs to do. Pseudocode is blueprints essentially. When you write pseudocode you capitalise the instruction. You write the programme out in statements – You can then convert that into code very quickly. More likely to land a job if you use and logically write pseudocode. You can debug pseudocode very quickly. Very useful. Coding jobs – often a test. Most people run out of time in the tests when writing the code, however if you have written out the pseudocode first, you have shown your plan for writing the code. Pseudocode specifically breaks down each command. It is important that we give specific instructions, every little detail is needed. We cannot rely on assumed knowledge when using commands.
    Misc.
A string is a collection of characters Booleans are true or false An empty string is false. Zero is false. an undefined variable and a variable with null is false. NaN – Not a number – returns as false
